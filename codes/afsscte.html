Introduction

Developers nowadays have much more access to sharable resources to accelerate their software development. Open source code repositories such as Github is one of the most popular one. However, a lack of documentation of majority open projects/codes in Github hinder their reusability. We propose a method aiming at improving the comprehension of source code by extracting informative topics from source code. The method automatically selects the combinations of types of terms (i.e., features), such as 'method name' and 'class name', that are most contributory to source code topic extraction. Specifically, it conducts an exhaustive search on all the feature combinations. The search is guided by the performance of topic extraction. We adopt two topic extraction algorithms from natural language processing and propose a method that leverages word embedding. A system has been developed to realize the topic extraction and the forward feature selection algorithms. It allows flexible parameter settings and visualizes the whole working process and results. It is language independent that can be applied to projects implemented by multiple programming languages.
